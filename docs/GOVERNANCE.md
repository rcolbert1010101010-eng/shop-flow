# Documentation Governance

## Definition of Done
A change is "done" only when the related documentation is updated and accurate for the current UI and behavior.

## Required Updates
- Any behavior change in a module requires a bump to the module's `updatedAt` in `src/help/docsRegistry.ts`.
- Docs must reflect actual UI behavior; avoid speculative features.

## Review Cadence
- Review documentation monthly to confirm accuracy and update dates.
